## 🌈 The 7-Layer Color-Coded Execution Logic Explained
To ensure absolute telemetry tracking upon the Ledger, all PiRC-2 transactions map to this specific color function:
- 🟠 **[Layer 1] OP_ORANGE (Registration):** Merchant accounts initialize the base parameter (price, lock-time, frequency) onto the blockchain.
- 🟡 **[Layer 2] OP_YELLOW (Subscription):** Subscriber keys cryptographically sign the mandate, locking into the recurring ledger constraints.
- 🔵 **[Layer 3] OP_BLUE (Boundary Extension):** Safely expands the periodic boundary allowance of the subscription without terminating it.
- 🟢 **[Layer 4] OP_GREEN (Omnipotent Settlement):** The bulk merchant extraction cycle where tokens are officially moved from subscribers to merchants.
- 🟣 **[Layer 5] OP_PURPLE (State Toggle):** A decentralized killswitch that enables/disables the `auto-renew` state variables.
- 🔴 **[Layer 6] OP_RED (Terminal Cancellation):** Hard-terminates the subscription logic.
- 🏅 **[Layer 7] OP_GOLD (Integrity Verification):** Queries the global protocol version, establishing total simulation readout.
